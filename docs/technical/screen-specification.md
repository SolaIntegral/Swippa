# Visuy Cast - 画面仕様書

## 1. 画面概要

### 1.1 システム名
**Swippa**

### 1.2 画面構成
1. **オンボーディング画面**
2. **メイン画面（OFFモード：公認サボり）**
3. **メイン画面（ONモード：集中）**
4. **タスク追加画面**
5. **スケジュール画面**

```
オンボーディング → メイン画面（OFF） ←→ メイン画面（ON）
     ↓                    ↓                    ↓
  初期設定           タスク追加画面        スケジュール画面
     ↓                    ↓                    ↓
  完了                 保存完了              ダッシュボード
```

## 2. 画面詳細仕様

### 2.1 オンボーディング画面

**画面ID**: S001  
**画面名**: オンボーディング画面  
**優先度**: 高  

**画面目的**
ユーザーの「面倒くさい」という最大の離脱リスクを突破し、アプリの価値を体験してもらう

**画面要素**

#### 2.1.1 ステップ1: アプリ紹介
- **ヘッダー**: Visuy Cast ロゴ
- **メインコンテンツ**: 
  - アプリのコンセプト説明
  - 「AIがあなたのやる気の波を管理します」
- **ナビゲーション**: 「次へ」ボタン

#### 2.1.2 ステップ2: なぞり入力チュートリアル
- **ヘッダー**: ステップ表示（2/4）
- **メインコンテンツ**:
  - 「週間スケジュールを指でなぞって登録」
  - インタラクティブなグリッド表示
  - 指のアニメーションガイド
- **ナビゲーション**: 「試してみる」ボタン

#### 2.1.3 ステップ3: 感情マトリクス説明
- **ヘッダー**: ステップ表示（3/4）
- **メインコンテンツ**:
  - 「達成感×抵抗感でAIが最適な提案をします」
  - 2軸グラフのサンプル表示
  - 入力例の説明
- **ナビゲーション**: 「理解しました」ボタン

#### 2.1.4 ステップ4: 最初のタスク登録
- **ヘッダー**: ステップ表示（4/4）
- **メインコンテンツ**:
  - 「まずはタスクを1つ登録してみましょう」
  - 簡易タスク入力フォーム
- **ナビゲーション**: 「登録して完了」ボタン

**UI仕様**
- **レイアウト**: 縦スクロール対応
- **アニメーション**: ステップ間のスムーズな遷移
- **色**: ブランドカラー（Primary Blue #3498db）
- **フォント**: システムフォント、読みやすいサイズ

**根拠・参考資料**
- リサーチ結果「オンボーディングUI事例」: プログレッシブオンボーディングが有効
- ペルソナ分析: 「入力が面倒くさい」という最大のペインポイントを解決

**エラーハンドリング**
- ネットワークエラー: オフライン時のローカル保存
- 入力エラー: リアルタイムバリデーション

### 2.2 メイン画面（OFFモード：公認サボり）

**画面ID**: S002  
**画面名**: メイン画面（OFFモード）  
**優先度**: 高  

**画面目的**
デフォルトの画面。ユーザーに「休んでいい」という許可を与え、起動時の心理的ハードルを下げる

**画面要素**

#### 2.2.1 ヘッダー
- **キャラクター**: 休憩中の様子（だらけている、寝ている）
- **モード表示**: 「OFFモード」の表示
- **時間表示**: 現在時刻

#### 2.2.2 メインコンテンツ
- **キャラクター**: 大きなキャラクター表示（休憩中）
- **メッセージ**: 「一緒に休みましょう」「今は休憩時間です」
- **背景**: 寒色系（青・緑）のグラデーション

#### 2.2.3 アクションエリア
- **モード切替ボタン**: 「ONモードにする」「タスクを始める」
- **タスク登録FAB**: 右下の「+」ボタン（常時表示）

**UI仕様**
- **レイアウト**: 中央配置、余白を多めに取る
- **色**: 寒色系（#3498db, #2ecc71）
- **アニメーション**: キャラクターの呼吸アニメーション
- **タッチエリア**: 最小48dp以上

**根拠・参考資料**
- コンセプト: 「公認サボり」機能の実装
- ペルソナ分析: 罪悪感の排除が重要

**エラーハンドリング**
- モード切替エラー: 手動での再試行
- FABタップエラー: タスク追加画面への遷移失敗時の再試行

### 2.3 メイン画面（ONモード：集中）

**画面ID**: S003  
**画面名**: メイン画面（ONモード）  
**優先度**: 高  

**画面目的**
ユーザーの「頑張りたい」意志をサポートし、タスク実行（スワイプ）に集中させる

**画面要素**

#### 2.3.1 ヘッダー
- **キャラクター**: やる気を出している様子（応援している）
- **モード表示**: 「ONモード」の表示
- **タイマー**: 集中時間のカウントダウン（例：25:00）

#### 2.3.2 メインコンテンツ
- **AI提案タスクカード**: 
  - タスク名
  - 締切
  - 感情アイコン（達成感・抵抗感）
  - スワイプ指示（「今やる！」「AIに任せる」）
- **キャラクター**: 応援している様子
- **背景**: 暖色系（オレンジ・赤）のグラデーション

#### 2.3.3 アクションエリア
- **スワイプエリア**: タスクカードの左右スワイプ
- **モード切替ボタン**: 「OFFモードに戻る（サボる）」
- **タスク登録FAB**: 右下の「+」ボタン（常時表示）

**UI仕様**
- **レイアウト**: タスクカードを中央に大きく配置
- **色**: 暖色系（#e74c3c, #f39c12）
- **アニメーション**: スワイプ時の滑らかなアニメーション
- **スワイプ感度**: 適切なスワイプ距離の設定

**根拠・参考資料**
- リサーチ結果「スワイプUIの適用事例」: 操作性が高く評価されている
- コンセプト: AIによる能動的なタスク提案

**エラーハンドリング**
- スワイプ認識エラー: タップ操作での代替
- AI推論エラー: フォールバックタスクの表示
- タイマーエラー: 手動での時間設定

### 2.4 タスク追加画面

**画面ID**: S004  
**画面名**: タスク追加画面  
**優先度**: 高  

**画面目的**
ユーザーが「面倒くささ」を感じる前に、タスク情報をシステムに放り込める

**画面要素**

#### 2.4.1 ヘッダー
- **タイトル**: 「タスクを追加」
- **戻るボタン**: 前の画面に戻る
- **保存ボタン**: タスクを保存

#### 2.4.2 メインコンテンツ
- **タスク名入力**:
  - ラベル: 「タスク名」
  - 入力欄: テキスト入力
  - プレースホルダー: 「例：レポート作成」
- **締切選択**:
  - ラベル: 「締切」
  - 入力欄: 日時選択
  - カレンダー: 直感的な日時選択UI
- **感情マトリクス**:
  - ラベル: 「このタスクへの気持ち」
  - 2軸グラフ: 達成感×抵抗感
  - 説明: 「AIが最適な提案をするために必要です」

#### 2.4.3 フッター
- **保存ボタン**: タスクを保存
- **キャンセルボタン**: 入力をキャンセル

**UI仕様**
- **レイアウト**: 縦スクロール対応
- **入力欄**: 適切なサイズ、フォーカス時のハイライト
- **感情マトリクス**: インタラクティブな2軸グラフ
- **バリデーション**: リアルタイムバリデーション

**根拠・参考資料**
- ペルソナ分析: 「入力が面倒くさい」という最大のペインポイントを解決
- リサーチ結果「感情×タスクのUI/UX事例」: 感情入力が有効

**エラーハンドリング**
- 入力不備: 必須項目未入力時のエラーメッセージ
- 締切過去: 過去の日時選択時の警告
- 保存エラー: ネットワークエラー時の再試行

**画面要素**

#### 2.6.1 ヘッダー

- **更新ボタン**: データの更新

#### 2.6.3 フッター

**UI仕様**
- **レイアウト**: カード形式での情報表示
- **色**: ポジティブな色合い（緑・青）
- **グラフ**: 分かりやすい可視化
- **アニメーション**: 数値のカウントアップアニメーション

**根拠・参考資料**

**エラーハンドリング**
- データ不足: 十分なデータがない場合のメッセージ
- 分析エラー: デフォルト表示での対応
- グラフ表示エラー: 数値のみでの表示

## 3. 画面間遷移

### 3.1 遷移ルール
- **オンボーディング**: 初回起動時のみ表示
- **メイン画面**: アプリの中心画面、常にアクセス可能
- **タスク追加**: 任意の画面からFABでアクセス
- **スケジュール**: メイン画面からナビゲーションでアクセス
- **ダッシュボード**: メイン画面からナビゲーションでアクセス

### 3.2 ナビゲーション
- **ボトムナビゲーション**: メイン・スケジュール・ダッシュボード
- **FAB**: 全画面でタスク追加
- **戻るボタン**: 各画面のヘッダーに配置
