# Visuy Cast - 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
**Swippa (ヴィスイ キャスト)**

### 1.2 プロジェクト目的
高い向上心を持つが、計画倒れや先延ばし癖（「完璧主義」×「サボりがち」）に悩み、疲弊と罪悪感の悪循環に陥っている大学生の課題を解決する。

### 1.3 ターゲットペルソナ
**張杉 翔（はりすぎ しょう）**
- 20歳、情報工学部2年生
- 完璧主義だが本質的には面倒くがり（サボりがち）
- 頑張りたい気持ちと行動できない現実のギャップに悩む

## 2. 解決する課題

### 2.1 主要課題
1. **キャパシティ把握の欠如**: 自身のキャパシティ（HP）を把握できないままタスク過多に陥る
2. **疲労による行動障壁**: 疲労により「タスクに着手できない」行動障壁が発生
3. **罪悪感の悪循環**: 先延ばしによる罪悪感が、さらなる行動障壁を生む「向上心からくる疲弊の悪循環」
4. **信頼・成長機会の損失**: 結果として、周囲の信頼や自己の成長機会を失ってしまう

### 2.2 ペインポイント
- タスクの存在自体を忘れ、締め切り直前に気づく
- 頑張りたい気持ちと行動できない現実のギャップ
- 従来のツールの「入力が面倒くさい」こと
- 「サボってしまった」罪悪感から解放されたい
- 「今サボっても大丈夫」というお墨付きが欲しい

## 3. 価値提案

### 3.1 独自の価値提案
単なる「受け身」のスケジュール管理ツールではなく、AIがユーザーの「やる気の波」や「キャパシティ」を能動的に管理する「パートナー」。

### 3.2 コアコンセプト
**「公認サボり＆集中プランナー」機能**
- AIの計算に基づき、罪悪感ゼロの「サボり時間」を提案
- サボってしまっても自動でスケジュールを再調整

### 3.3 差別化ポイント
- **vs Googleカレンダー**: 既存ツールは「受け身」。Visuy CastはAIが「能動的」にスケジュールを提案し、警告する
- **vs Notion/Todoist**: 既存ツールは「合理的（時間・優先度）」で管理する。Visuy Castは「感情的（達成感×抵抗感）」な側面も考慮して管理する

## 4. 機能要件

### 4.1 スケジュール・タスク管理機能

#### 4.1.1 固定予定（ブロック）の登録
- **目的**: 大学の授業やバイトなど、時間が固定された予定を登録
- **要件**: 週間のグリッドを「指でなぞる」だけで直感的に登録できる
- **根拠**: リサーチ結果「なぞり入力UIの競合調査」で操作性が高く評価されている

#### 4.1.2 タスク（To-Do）の登録
- **目的**: 「レポート作成」など、締切はあるが実行時間が未定のタスクを登録
- **要件**: 思いついた瞬間に忘れないよう、どの画面からでも1タップ（FAB）で登録できる
- **根拠**: ペルソナ分析で「タスクの存在自体を忘れる」が最大のペインポイント

#### 4.1.3 感情マトリクスの入力
- **目的**: タスク登録時に、そのタスクに対する「達成感（やりたい）」と「抵抗感（やりたくない）」を2軸グラフで入力
- **要件**: AIが「やる気の波」を推測する材料を提供する
- **根拠**: リサーチ結果「感情×タスクのUI/UX事例」で感情入力が有効とされている

### 4.2 AIによる能動的サポート機能

#### 4.2.1 モード管理（ON/OFF）
- **目的**: ユーザーの状態を「ON（集中）」と「OFF（公認サボり）」の2モードで管理
- **要件**: 「サボる」ことをシステムが公認し、メリハリをつけることで罪悪感を排除する
- **根拠**: リサーチ結果「戦略的休息の効果」で計画的休息が生産性向上に寄与することが証明されている

#### 4.2.2 AIタスク提案（スワイプUI）
- **目的**: ONモード時、AIがユーザーの「感情マトリクス」と「残キャパシティ」に基づき、今やるべきタスクを1つだけ提案
- **要件**: マッチングアプリのようなスワイプ操作で直感的に選択
- **根拠**: リサーチ結果「スワイプUIの適用事例」で操作性が高く評価されている

#### 4.2.3 AI自動リスケジュール（AIに任せる）
- **目的**: 提案されたタスクを「今やりたくない」場合、スワイプ操作で「AIに任せる」を選択
- **要件**: AIが自動で最適な時間（例：キャパが回復する別の日時）に再配置する
- **根拠**: リサーチ結果「スワイプUIの適用事例」で「後回し」の罪悪感を「リスケ依頼」に変換する設計が有効

#### 4.2.4 自動フォールバック（自動サボり検知）
- **目的**: ONモード中に一定時間タスクに着手しない（放置する）と、AIが「今は休憩が必要」と判断し、自動でOFFモード（公認サボり）に切り替える
- **要件**: 「サボってしまった…」と自己嫌悪する前に、「AIが『今は休憩時間』と判断した」と認識をすり替える

### 4.3 振り返り・モチベーション維持機能

#### 4.3.1 ポジティブ・ダッシュボード
- **目的**: 行動データを「失敗」としてではなく、客観的な「タイプ」や「質的成長」としてポジティブに可視化
- **要件**: 「またサボった」という自己嫌悪ではなく、「自分は夜型だ」「難しいタスクをクリアした」という自己効力感を得る
- **根拠**: リサーチ結果「行動パターンダッシュボードの事例」でポジティブな可視化が有効とされている

#### 4.3.2 共感者キャラクター
- **目的**: ONモード（応援）とOFFモード（一緒に休む）で様子が変わり、ユーザーの感情に寄り添う
- **要件**: 「監視者」ではなく「共感者」として振る舞う
- **根拠**: リサーチ結果「ゲーミフィケーションとペルソナ適性」で過度なプレッシャーは逆効果とされている

## 8. リスクと対策

### 8.1 想定リスク
1. **入力の面倒くささ**: ユーザーが初期設定で離脱するリスク
2. **罪悪感の増幅**: ゲーミフィケーションが逆にプレッシャーになるリスク
3. **機能の複雑さ**: 多機能によりユーザーが混乱するリスク

### 8.2 対策
1. **プログレッシブオンボーディング**: 最小限の入力から始める
2. **共感者キャラクター**: 監視者ではなく共感者として設計
3. **シンプルなUI**: 4つの主要機能を一画面で直感的に操作
