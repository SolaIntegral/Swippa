# Swippa - Figmaデザイン仕様書

## 📋 プロジェクト概要

### プロジェクト名
**Swippa**（旧名: Visuy Cast）

### ターゲットユーザー
**ペルソナ: 張杉 翔（はりすぎ しょう）**
- 20歳、情報工学部2年生
- **特性**: 高い向上心を持つが、計画倒れや先延ばし癖に悩む
- **課題**: 「頑張りたい気持ち」と「行動できない現実」のギャップ、罪悪感の悪循環
- **ニーズ**: 罪悪感なく挑戦と休息を両立させたい

### コアコンセプト
**「公認サボり＆集中プランナー」**

AIがユーザーの「やる気の波」や「キャパシティ」を能動的に管理し、タスク実行だけでなく「サボること」すらも計画に組み込み、罪悪感なく挑戦と休息を両立させる。

### デザインの方向性
**Duolingo風の楽しく魅力的なUI**
- カラフルな配色（グラデーション）
- 丸みを帯びたデザイン（角丸、カード型）
- キャラクターの活用
- アニメーション/マイクロインタラクション
- ゲーミフィケーション要素（バッジ、進捗バー）

- **小**: 8px（小さな要素）
- **中**: 16px（カード、ボタン）

- **特大**: 24px（モーダルコンテナ）
- **完全**: 50%（円形ボタン、アバター）

### シャドウ

- **Small**: `0 2px 8px rgba(0,0,0,0.05)`
- **Medium**: `0 4px 12px rgba(0,0,0,0.08)`
- **Large**: `0 8px 24px rgba(0,0,0,0.1)`
- **XLarge**: `0 12px 32px rgba(0,0,0,0.15)`
- **Colored**: `0 4px 12px rgba(52, 152, 219, 0.3)`（プライマリカラーのシャドウ）

### アニメーション

#### トランジション
- **標準**: `0.3s ease` または `0.3s cubic-bezier(0.4, 0, 0.2, 1)`
- **高速**: `0.15s ease`
- **低速**: `0.5s ease`

#### キーフレームアニメーション
- **Pulse**: スケールが1.0 ↔ 1.05で変化（タイマー、バッジ）
- **Bounce**: 上下に5px移動（スワイプヒント、アイコン）
- **Float**: 上下に8px移動（FAB、キャラクター）
- **FadeIn**: 透明度0 → 1（モーダル）
- **SlideUp**: Y軸50px移動 + スケール0.9 → 1.0（モーダル）

## 🎯 デザイン原則

### 1. Duolingo風の要素
- **カラフルな配色**: グラデーションを多用
- **丸みを帯びたデザイン**: 角丸を大きく（16px以上）
- **キャラクターの活用**: 各画面で共感的なキャラクター表示
- **アニメーション**: マイクロインタラクションで楽しさを演出
- **ゲーミフィケーション**: 進捗バー、バッジ、アニメーション

### 2. モバイルファースト
- **タッチターゲット**: 最小48px
- **スクロール**: 縦スクロール優先
- **ナビゲーション**: フッター固定
- **レスポンシブ**: スマホ（0-767px）を最優先

### 3. アクセシビリティ
- **コントラスト**: 4.5:1以上
- **フォーカス**: 明確なアウトライン
- **セマンティック**: 適切なHTML構造

### 4. パフォーマンス
- **アニメーション**: 60fps維持
- **トランジション**: 0.3s以下
- **画像**: 最適化、遅延読み込み
